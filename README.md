# PRD文档管理

本文件夹用于存放各产品线的PRD（Product Requirements Document，产品需求文档）。

## 文件夹结构

```
PRD/
├── README.md           # 本说明文件
├── 专业助手/          # 专业助手产品线PRD文档（前端页面）
├── 科研助手/          # 科研助手产品线PRD文档（前端页面）
├── 患管助手/          # 患管助手产品线PRD文档（前端页面）
├── IMDT/              # IMDT产品线PRD文档（前端页面）
└── 管理后台/          # 管理后台PRD文档（后台web页面）
```

## 产品线说明

### 1. 专业助手
- **定位**：为医生提供专业医疗知识辅助和临床决策支持
- **PRD文档**：存放在 `专业助手/` 文件夹中

### 2. 科研助手
- **定位**：服务医生，提供科研辅助功能
- **PRD文档**：存放在 `科研助手/` 文件夹中

### 3. 患管助手
- **定位**：服务院内医生，提供患者管理功能和健康管理功能
- **PRD文档**：存放在 `患管助手/` 文件夹中

### 4. IMDT
- **定位**：区域性MDT协作平台
- **PRD文档**：存放在 `IMDT/` 文件夹中

### 5. 管理后台
- **定位**：统一的管理后台系统，包含角色配置、SOP配置、工具配置、权限管理、数据统计等功能
- **功能模块**：
  - 入口二维码管理
  - 渠道管理
  - SOP管理
  - 企业微信配置
  - 会议管理
  - 参会人员管理
  - 智能体管理
  - 小程序管理
  - 统一注册管理
  - 基础报表
  - 统计分析
  - 系统管理
- **PRD文档**：存放在 `管理后台/` 文件夹中

## PRD文档规范

### 文档命名规范
建议使用以下命名格式：
- `产品线名称_PRD_v版本号_日期.md`
- 例如：`科研助手_PRD_v1.0_20250115.md`

### PRD文档结构建议
1. **产品概述**
   - 产品定位
   - 目标用户
   - 核心价值

2. **市场分析**
   - 市场背景
   - 竞品分析
   - 用户需求

3. **功能需求**
   - 功能列表
   - 功能详细说明
   - 优先级划分

4. **非功能需求**
   - 性能要求
   - 安全要求
   - 兼容性要求

5. **技术方案**
   - 技术架构
   - 关键技术点
   - 技术选型

6. **开发计划**
   - 里程碑规划
   - 资源需求
   - 风险评估

7. **验收标准**
   - 功能验收标准
   - 性能验收标准
   - 用户体验验收标准
